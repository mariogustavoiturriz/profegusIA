Paso 1. Declarás las dos variables al principio del código.

import requests # ── 1. Las variables que VOS elegís ────────────────────────────── personaje = "Albert Einstein" tema = "los videojuegos" API_KEY = "gsk_PEGA_TU_CLAVE_ACÁ" # ── 2. Armamos la pregunta CON f-string ────────────────────────── # Las variables van adentro de llaves { } dentro del string pregunta = f"Actuá como {personaje} y explicame qué pensás sobre {tema}." # Resultado: "Actuá como Albert Einstein y explicame qué pensás sobre los videojuegos." # Si cambiás personaje o tema arriba, el texto se actualiza solo. # ── 3. La conexión (igual que siempre) ─────────────────────────── MODEL = "llama-3.3-70b-versatile" URL = "https://api.groq.com/openai/v1/chat/completions" headers = {"Authorization": f"Bearer {API_KEY}", "Content-Type": "application/json"} payload = { "model": MODEL, "messages": [ {"role": "user", "content": pregunta} # ← la variable, no el texto fijo ] } response = requests.post(URL, headers=headers, json=payload) data = response.json() print(data["choices"][0]["message"]["content"])
¿Por qué funciona así? El f-string arma el texto antes de enviarlo. Cuando Python ejecuta la línea del f-string, reemplaza {personaje} por el valor de la variable y {tema} por el suyo. El resultado es un texto normal — como si lo hubieras escrito a mano — y eso va al payload. La ventaja: si querés cambiar el personaje o el tema, solo tocás las variables del principio.

1 GLOSARIO — CONCEPTOS NUEVOS DE HOY

Antes de tocar el código, tres conceptos que vamos a necesitar en toda la clase.

JSON de Respuesta Cuando el modelo responde, el servidor no manda solo el texto. Manda un paquete completo de información sobre toda la llamada: el nombre del modelo que respondió, cuántos tokens se consumieron, si terminó bien o fue interrumpido, y más. En C5 leímos solo una pequeña parte de ese paquete. Hoy exploramos todo lo que hay adentro.
Temperature Un número entre 0.0 y 2.0 que se agrega al payload. Controla qué tan predecible o creativo es el modelo al elegir el siguiente token. No cambia lo que el modelo sabe — cambia cómo elige entre las opciones posibles.
  • 0.0 → Siempre elige el token más probable. Respuestas consistentes, casi idénticas si ejecutás varias veces.
  • 0.7 → Un punto medio. La respuesta cambia un poco cada vez que ejecutás, pero se mantiene lógica y tiene sentido. Es el valor por defecto si no le ponés número.
  • 1.5+ → Alta variación. Puede tirar ideas muy creativas o directamente decir cualquier locura que no tiene nada que ver.
Alucinación Cuando el modelo genera texto convincente pero incorrecto, impreciso o directamente inventado. No es un error del programa. Es una consecuencia del mecanismo: el modelo predice lo que estadísticamente "suena bien", aunque no sea verdad. No tiene acceso a internet, no verifica nada. Solo predice el siguiente token.

2 LA RESPUESTA COMPLETA — NAVEGANDO EL JSON

En C5 usamos esta línea para ver solo el texto:

print(data["choices"][0]["message"]["content"])

Pero data tiene mucho más. Antes de esa línea, agregá:

print(data) # imprimí el paquete completo y mirá todo lo que manda el servidor

El servidor manda algo parecido a esto (simplificado para que sea legible):

{ "id": "chatcmpl-abc123xyz", "model": "llama-3.3-70b-versatile", "choices": [ { "index": 0, "message": { "role": "assistant", "content": "El villano principal de Batman es El Joker..." }, "finish_reason": "stop" } ], "usage": { "prompt_tokens": 18, "completion_tokens": 142, "total_tokens": 160 } }

¿Qué significa cada parte?

  • "id": Identificador único de esta llamada. Útil para rastrear errores: si algo falla, este id te dice exactamente qué llamada fue.
  • "model": Confirma qué modelo respondió. Útil para verificar que usaste el que pediste.
  • "choices": Es siempre una lista [ ] aunque casi siempre tenga un solo elemento. La API está diseñada para devolver varias respuestas al mismo tiempo si se lo pedís (con el parámetro n). Por defecto devuelve una sola, pero siempre envuelta en una lista. Por eso accedemos con [0].
  • "message": El diccionario con la respuesta del modelo. Siempre tiene "role": "assistant" y "content" con el texto.
  • "finish_reason": Por qué paró el modelo. "stop" significa que terminó normalmente. "length" significa que se cortó por llegar al límite de tokens.
  • "usage": La cuenta exacta de tokens de esta llamada. Muy importante si el servicio es pago.
¿Qué son prompt_tokens y completion_tokens? prompt_tokens: los tokens de tu pregunta (lo que enviaste vos). completion_tokens: los tokens de la respuesta del modelo (lo que recibiste). total_tokens: la suma de los dos. En servicios de pago como OpenAI, se cobra por total_tokens — cuanto más larga la pregunta, más cara la llamada.

Cómo acceder a cada parte por separado

En lugar de imprimir todo junto, podés navegar el diccionario con las claves que ya conocés:

import requests API_KEY = "gsk_PEGA_TU_CLAVE_ACÁ" MODEL = "llama-3.3-70b-versatile" URL = "https://api.groq.com/openai/v1/chat/completions" headers = {"Authorization": f"Bearer {API_KEY}", "Content-Type": "application/json"} payload = { "model": MODEL, "messages": [{"role": "user", "content": "Escribí un chiste corto."}] } response = requests.post(URL, headers=headers, json=payload) data = response.json() # Accedemos a cada parte del JSON por separado print("── El texto de la respuesta:") print(data["choices"][0]["message"]["content"]) print("\n── El modelo que respondió:") print(data["model"]) print("\n── Por qué paró el modelo:") print(data["choices"][0]["finish_reason"]) print("\n── Los tokens consumidos:") print(data["usage"])

3 EL PARÁMETRO TEMPERATURE — EXPERIMENTO

¿Recordás la frase clave de C5? "El LLM predice el siguiente token más probable." Cuando el modelo está por escribir la siguiente palabra, tiene en mente una lista de candidatos posibles, cada uno con un porcentaje de probabilidad. El parámetro temperature controla qué tan arriesgado es al elegir de esa lista.

Temperature = 0.0 — frío Siempre elige el token con mayor probabilidad. El resultado es casi determinista: si ejecutás el mismo código tres veces, la respuesta es prácticamente idéntica. Útil cuando necesitás consistencia: resúmenes, traducciones, extracción de datos.
Temperature = 0.7 — templado El modelo puede elegir tokens un poco menos probables. Las respuestas varían cada vez que ejecutás el código, pero siempre mantienen la lógica y tienen sentido. Es el valor por defecto. Bueno para conversaciones, redacción, chatbots.
Temperature = 1.5+ — caliente El modelo puede elegir tokens poco probables con frecuencia. Las respuestas son muy variadas: a veces sorprendentes, a veces sin sentido. Útil para lluvia de ideas o tareas creativas. No recomendado cuando necesitás precisión.

¿Cómo se agrega al código?

Solo hay que agregar una línea al payload:

payload = { "model": MODEL, "temperature": 0.7, # ← esta línea es nueva. Podés cambiar el número. "messages": [ {"role": "user", "content": "Tu pregunta acá"} ] }

Copiá este código que hace el experimento completo de una sola vez, con las tres temperatures:

import requests API_KEY = "gsk_PEGA_TU_CLAVE_ACÁ" MODEL = "llama-3.3-70b-versatile" URL = "https://api.groq.com/openai/v1/chat/completions" headers = {"Authorization": f"Bearer {API_KEY}", "Content-Type": "application/json"} prompt = "Describí la ciudad de Ushuaia en una sola oración." temperaturas = [0.0, 0.7, 1.5] for temp in temperaturas: payload = { "model": MODEL, "temperature": temp, "messages": [{"role": "user", "content": prompt}] } response = requests.post(URL, headers=headers, json=payload) data = response.json() print(f"──── Temperature {temp} ──────────────────────────────") print(data["choices"][0]["message"]["content"]) print()

4 LA ALUCINACIÓN — POR QUÉ EL MODELO INVENTA

El modelo no busca información. Predice tokens. Eso significa que cuando le hacés una pregunta, no consulta ninguna base de datos, no va a internet, no verifica nada. Solo genera el texto que estadísticamente tiene más probabilidad de seguir al tuyo.

La consecuencia directa: si el texto más probable estadísticamente es incorrecto, el modelo lo genera igual — y con la misma confianza con la que genera texto correcto. No sabe que está equivocado.

¿Cuándo es más probable que alucine?
  • Cuando preguntás por datos numéricos precisos: fechas exactas, cantidades, estadísticas.
  • Cuando preguntás sobre información local o específica que tiene poca presencia en internet.
  • Cuando preguntás sobre eventos recientes (el modelo tiene una "fecha de corte" en su entrenamiento y no sabe qué pasó después).
  • Cuando una pregunta tiene una respuesta que "suena lógica" pero no es real.
¿Cuándo suele NO alucinar?
  • Conocimiento científico consolidado: fórmulas, leyes físicas, conceptos matemáticos.
  • Hechos históricos ampliamente documentados en todo internet.
  • Tareas donde no necesita recordar datos: resumir, traducir, reescribir, explicar.
