import requests # ── 1. Las variables que VOS elegís ────────────────────────────── personaje = "Albert Einstein" tema = "los videojuegos" API_KEY = "gsk_PEGA_TU_CLAVE_ACÁ" # ── 2. Armamos la pregunta CON f-string ────────────────────────── # Las variables van adentro de llaves { } dentro del string pregunta = f"Actuá como {personaje} y explicame qué pensás sobre {tema}." # ── 3. La conexión (igual que siempre) ─────────────────────────── MODEL = "llama-3.3-70b-versatile" URL = "https://api.groq.com/openai/v1/chat/completions" headers = {"Authorization": f"Bearer {API_KEY}", "Content-Type": "application/json"} payload = { "model": MODEL, "messages": [ {"role": "user", "content": pregunta} # ← la variable, no el texto fijo ] } response = requests.post(URL, headers=headers, json=payload) data = response.json() print(data["choices"][0]["message"]["content"])
¿Por qué funciona así? El f-string arma el texto antes de enviarlo. Cuando Python ejecuta la línea del f-string, reemplaza {personaje} por el valor de la variable y {tema} por el suyo. El resultado es un texto normal y eso va al payload. La ventaja: si querés cambiar el personaje o el tema, solo tocás las variables del principio.

1 GLOSARIO — CONCEPTOS NUEVOS DE HOY

Antes de tocar el código, tres conceptos que vamos a necesitar en toda la clase.

JSON de Respuesta Cuando el modelo responde, el servidor no manda solo el texto. Manda un paquete completo de información sobre toda la llamada: el nombre del modelo que respondió, cuántas unidades se procesaron, y metadatos técnicos.
Temperature Es un número entre 0.0 y 2.0 que se agrega al payload. Funciona como un regulador que controla qué tan arriesgado o creativo es el modelo al elegir cada palabra de la respuesta.
Alucinación Cuando el modelo genera texto convincente pero incorrecto, impreciso o directamente inventado. No es un error del programa. Es una consecuencia del mecanismo: el modelo predice lo que estadísticamente "suena bien", aunque no sea verdad.

2 ACTIVIDAD 1: NAVEGANDO EL JSON

En C5 usamos esta línea para ver solo el texto:

print(data["choices"][0]["message"]["content"])

Pero data tiene mucho más. Antes de esa línea, agregá:

print(data) # imprimí el paquete completo y mirá todo lo que manda el servidor

¿Qué significa cada parte?

  • "id": Identificador único de esta llamada. Útil para rastrear errores técnicos.
  • "model": Confirma qué modelo respondió realmente.
  • "choices": Es una lista [ ] que contiene la respuesta. Aunque suele haber una sola, siempre viene en lista; por eso usamos [0].
  • "message": El diccionario con la respuesta del modelo (texto y rol).
  • "finish_reason": Por qué paró el modelo. "stop" significa que terminó bien.
  • "usage": Es el contador de "gasto" o volumen. Nos dice cuántas unidades de texto (tokens) se procesaron en total en la llamada.

Cómo acceder a cada parte por separado

3 ACTIVIDAD 2: EL PARÁMETRO TEMPERATURE — EXPERIMENTO

¿Recordás la frase clave de C5? "El LLM predice el siguiente token más probable." El parámetro temperature controla qué tan arriesgado es el modelo al elegir de la lista de opciones posibles.

Temperature = 0.0 — frío Siempre elige la opción más probable. El resultado es casi idéntico cada vez que ejecutás. Útil para tareas que requieren precisión.
Temperature = 0.7 — templado Las respuestas varían cada vez que ejecutás el código, pero siempre mantienen la lógica y tienen sentido. Es el valor por defecto.
Temperature = 1.5+ — caliente El modelo elige opciones poco probables. Las respuestas son muy variadas y a veces sorprendentes o raras. Útil para tareas creativas.

¿Cómo se agrega al código?

Solo hay que agregar una línea al payload:

payload = { "model": MODEL, "temperature": 0.7, # ← esta línea es nueva. Podés cambiar el número. "messages": [ {"role": "user", "content": "Tu pregunta acá"} ] }

4 ACTIVIDAD 3: LAS ALUCINACIONES — POR QUÉ EL MODELO INVENTA

El modelo no busca información. Predice tokens. Eso significa que cuando le hacés una pregunta, no consulta ninguna base de datos real. Solo genera el texto que estadísticamente tiene más probabilidad de seguir al tuyo.

¿Cuándo es más probable que alucine?
  • Cuando preguntás por datos numéricos exactos o fechas muy precisas.
  • Cuando preguntás sobre información local (ej: de Ushuaia o Tierra del Fuego).
  • Cuando preguntás sobre eventos muy recientes.
