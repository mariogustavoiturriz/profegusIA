⚙️ FASE 1: DESARROLLO DEL CÓDIGO (Viernes - 80 min)

1 ARQUITECTURA DEL SOFTWARE

Creen un nuevo Cuaderno en Colab llamado TP1_Apellidos. El programa debe cumplir estrictamente con esta estructura:

Módulo 1: La Función (Backend)

Toda la lógica de conexión debe estar encapsulada para ser reutilizable.

  • Crear una función def consultar_ia(personaje, tema, temperatura):
  • Dentro de la función, armar el payload inyectando los parámetros. Usar f-strings para armar el prompt dinámico.
  • El Escudo: Envolver el requests.post y el manejo del JSON en un bloque try / except. Si la API falla, la función no debe romperse, debe retornar un mensaje de error amigable.
  • La función debe usar return para devolver exclusivamente el texto de la respuesta limpia de la IA. Opcional: imprimir por consola los tokens consumidos.

Módulo 2: Interfaz de Consola (Frontend)

Nada puede estar "hardcodeado" (escrito fijo en el código). Todo se pide en tiempo de ejecución:

  • Usar input() para pedir el Personaje y el Tema.
  • Usar input() para pedir el Nivel de Creatividad (Temperature). ¡Atención técnica! Este valor es un número con decimales, por lo que deben usar float() para convertir el texto ingresado por el usuario.
  • Llamar a su función pasándole estos tres datos, y hacer el print() final del resultado.
  • (Si son 2 o 3 integrantes, acá es donde aplican el while True y/o el historial).
🗣️ FASE 2: DOCUMENTACIÓN Y DEFENSA (Lunes - 120 min)

2 DOCUMENTACIÓN TÉCNICA Y QA

Debajo de su código, agreguen una Celda de Texto en Colab y respondan con lenguaje técnico, consensuado con su grupo:

  1. Alucinaciones: Expliquen qué es una Alucinación en Inteligencia Artificial y por qué el modelo inventa respuestas (basado en cómo predice tokens) en lugar de decir "no sé".
  2. Temperature: Si configuran la temperatura en 0.0, ¿cómo se comporta el modelo a nivel estadístico? ¿En qué casos usarían 0.0 y en qué casos 1.5?
  3. Arquitectura: Miren su propio código. ¿Qué línea exacta cambiarían si el día de mañana deciden que, en lugar de llama-3.3-70b-versatile, quieren usar el modelo mixtral-8x7b-32768?
  4. Seguridad: ¿Por qué es una mala práctica dejar la API_KEY escrita directamente en el código fuente en un proyecto real de la industria?
🎤 Preparación para la Defensa Oral:

El profesor pasará por los bancos validando los trabajos. Cada integrante deberá poder explicar una parte del código leyendo los comentarios que dejaron con el #. Si no pueden justificar por qué usaron un return, un try o un float, la nota grupal se verá afectada.
