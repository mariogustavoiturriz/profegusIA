0 EL PUNTO DE PARTIDA Y EL ERROR

Venimos de días sin clases. En la C4 aprendimos a usar try / except. ¿Por qué era tan importante?

Hasta ahora nuestro código vivía encerrado en nuestra compu. Hoy, Python va a salir a internet para conectarse a un servidor de IA. Cuando salís a la red, los fallos son reales: se corta el wifi, el servidor está caído o te rechazan el acceso. El try/except es nuestro escudo para evitar que el programa explote si algo de eso pasa.

1 GLOSARIO TÉCNICO

Para programar IA en el entorno profesional, la terminología debe ser exacta. Leé y asimilá estos conceptos antes de avanzar:

LLM (Large Language Model) Modelos entrenados con millones de textos. Su única función matemática es predecir cuál es la palabra más probable que sigue en una oración.
Token La IA no lee "palabras". Divide el texto en fragmentos más pequeños llamados tokens (aprox. 4 caracteres). Esto le permite transformar texto en números para hacer cálculos matemáticos.
API (Interfaz de Programación de Aplicaciones) Es como un "enchufe" que permite que nuestro código en Python se comunique con el servidor de una Inteligencia Artificial a través de internet, sin necesidad de saber cómo está construida la IA por dentro.
JSON (JavaScript Object Notation) Es un formato estándar mundial para enviar datos por internet. En Python, la respuesta JSON que nos llega se convierte automáticamente en el mismo formato de Diccionarios que ya aprendimos: pares de {"clave": valor}.
API Key (Tu credencial de acceso) Para evitar que alguien sature los servidores haciendo millones de preguntas, los sistemas de IA requieren autenticación. La API Key es una cadena de texto alfanumérica única (tu contraseña secreta). Si tu código no envía esta llave, el servidor rechaza la conexión con un error de "Acceso Denegado".

2 CONOCIENDO A GROQ Y TU API KEY

Para hacer nuestro programa, vamos a conectarnos a los servidores de una empresa llamada Groq. Es una plataforma gratuita y rapidísima que nos presta el acceso al "cerebro" de las IAs más potentes del mundo.

Para evitar saturaciones, Groq necesita saber quién sos antes de dejarte pasar. Sin tu propia llave de acceso (API Key), no podés programar hoy. Vamos a generarla:

4 LA ESTRUCTURA DE LA CONEXIÓN Y EL CÓDIGO BASE

Antes de ejecutar, tenemos que entender las dos partes fundamentales de datos que le enviamos al servidor de Groq: los headers y el payload.

A) Los Headers (El "sobre" de la carta)

headers = { "Authorization": f"Bearer {API_KEY}", "Content-Type": "application/json" }
  • "Authorization": Inyectamos nuestra API_KEY. Bearer significa que somos los portadores autorizados.
  • "Content-Type": Le avisa a Groq que el mensaje está en formato JSON.

B) El Payload (El contenido del mensaje)

payload = { "model": "llama-3.3-70b-versatile", "messages": [ {"role": "user", "content": "¿Qué es Python?"} ] }

Copiá el código base y reemplazá la línea 4 por tu clave real.
